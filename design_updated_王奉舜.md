# 上证 A 股出版传媒上市公司行情与主力资金流周报分析工具设计文档

## 1. 项目背景

中国 A 股市场中存在多家以出版、发行、传媒、文化内容服务为主营业务的上市公司。这类公司分布在出版传媒行业，既具有文化产业属性，也受到资本市场行情、行业政策、资金流向和市场情绪影响。

手动查看每家公司的开盘价、收盘价、涨跌幅、主力净流入等数据效率较低，也不利于长期观察行业板块变化。本项目计划开发一个轻量级 Python 数据分析工具，持续跟踪上证 A 股出版传媒类上市公司的关键行情和资金流数据，并每周自动生成一份结构化 Excel 周报。

本项目只用于课程实践、数据整理和趋势观察，不构成任何投资建议。

## 2. 目标用户

- 关注出版传媒行业上市公司表现的研究者或学习者。
- 希望定期整理 A 股出版传媒类公司行情数据的个人用户。
- 需要快速生成周度市场观察材料的内容分析人员。

## 3. 核心场景

用户希望每周查看一次上证 A 股出版传媒类上市公司的行情变化和资金流向。工具根据预设股票池获取每日数据，保存历史记录，并在每周生成一份 Excel 格式周报。

典型使用流程：

1. 用户在配置文件中确认需要跟踪的股票代码和名称。
2. 用户运行数据采集命令，工具获取最新交易日数据。
3. 工具保存开盘价、收盘价、涨跌幅、主力净流入等字段。
4. 每周末或指定时间，用户运行周报生成命令。
5. 工具读取本周历史数据，计算周涨跌幅、主力净流入、排名等指标。
6. 工具输出 `.xlsx` 周报，供用户查看、提交或继续编辑。

## 4. 产品形态

第一版采用 Python 本地数据分析工具形态，不开发复杂 Web 系统。

选择原因：

- Python 适合行情数据获取、清洗、统计和 Excel 报告生成。
- 第一版重点是数据跟踪和周报输出，不需要复杂交互界面。
- 本地脚本便于课程作业展示和后续扩展。
- 相比 Web 工具，开发成本更低，数据处理流程更清晰。

后续如需要展示效果，可以扩展为 Streamlit 或 Flask Web 页面。

## 6. 可接入股票行情 API 方案

本项目需要接通可用的股票行情数据接口，方便按股票代码实时或准实时调取特定股票信息。第一版建议采用“AkShare 优先、Tushare Pro 备选”的策略。

### 6.1 推荐方案：AkShare

AkShare 是 Python 金融数据接口库，适合本地数据分析项目。它提供东方财富等数据源的 A 股实时行情、历史行情和资金流向接口，不需要先搭建服务器。

可用接口：

| 数据需求 | AkShare 接口 | 说明 |
|---|---|---|
| A 股实时行情 | `stock_zh_a_spot_em()` | 返回沪深京 A 股实时行情数据，可按股票代码过滤特定股票 |
| 上证 A 股实时行情 | `stock_sh_a_spot_em()` | 返回沪 A 股实时行情数据，适合本项目股票池 |
| 个股历史日线 | `stock_zh_a_hist(symbol, period, start_date, end_date, adjust)` | 获取开盘价、收盘价、涨跌幅、成交量等日频数据 |
| 个股资金流 | `stock_individual_fund_flow(stock, market)` | 获取指定股票近 100 个交易日资金流数据 |
| 个股资金流排名 | `stock_individual_fund_flow_rank(indicator)` | 获取今日、3 日、5 日、10 日资金流排名 |

优点：

- 适合 Python 课程项目，安装和调用成本低。
- 能覆盖 A 股行情和东方财富资金流数据。
- `stock_zh_a_spot_em()` 可实现按特定股票代码获取最新行情。
- `stock_individual_fund_flow()` 可提供主力净流入相关字段。

限制：

- 免费数据源可能存在接口变动、访问频率限制或字段调整。
- 主力资金流是第三方平台算法结果，不是交易所官方原始字段。
- 实时行情依赖第三方页面或数据服务，不能保证用于真实交易决策。

示例调用：

```python
import akshare as ak

# 获取全市场实时行情后按代码过滤
spot_df = ak.stock_zh_a_spot_em()
target = spot_df[spot_df["代码"] == "601949"]

# 获取单只股票历史日线
hist_df = ak.stock_zh_a_hist(
    symbol="601949",
    period="daily",
    start_date="20260608",
    end_date="20260612",
    adjust=""
)

# 获取上海市场个股资金流
fund_df = ak.stock_individual_fund_flow(stock="601949", market="sh")
```

### 6.2 备选方案：Tushare Pro

Tushare Pro 提供较稳定的结构化股票数据接口，适合需要 token、权限和更稳定历史数据的场景。

可用接口：

| 数据需求 | Tushare Pro 接口 | 说明 |
|---|---|---|
| 股票列表 | `stock_basic` | 获取股票代码、名称、上市状态、行业等基础信息 |
| A 股日线行情 | `daily` | 获取开盘价、收盘价、涨跌幅、成交量、成交额 |
| 个股资金流 | `moneyflow` | 获取沪深 A 股资金流向数据，包括大单、特大单和净流入字段 |
| 交易日历 | `trade_cal` | 判断指定日期是否为交易日 |

优点：

- 接口字段稳定，适合长期保存历史数据。
- 提供交易日历，可减少非交易日采集错误。
- `moneyflow` 有明确的大小单与净流入字段。

限制：

- 需要注册 token，部分接口需要积分权限。
- 日线行情通常在交易日 15:00 至 16:00 间入库，实时性弱于行情软件类实时接口。
- 第一版如果只是课程演示，接入成本高于 AkShare。

示例调用：

```python
import tushare as ts

pro = ts.pro_api("your_token")

daily_df = pro.daily(
    ts_code="601949.SH",
    start_date="20260608",
    end_date="20260612"
)

moneyflow_df = pro.moneyflow(
    ts_code="601949.SH",
    start_date="20260608",
    end_date="20260612"
)
```

### 6.3 不作为第一版主数据源的方案

| 方案 | 适用性 | 不优先采用原因 |
|---|---|---|
| Alpha Vantage | 美股、外汇、全球市场较常见 | A 股覆盖和主力资金流不适合本项目 |
| Finnhub | 美股和全球金融数据较常见 | 对中国 A 股和主力资金流支持不足 |
| 券商交易 API | 可用于真实账户交易或行情 | 通常需要开户、权限和合规审查，超出课程项目范围 |
| 东方财富网页接口直连 | 可获得行情数据 | 接口稳定性、授权和反爬风险较高，建议通过 AkShare 间接调用 |

### 6.4 第一版数据源决策

第一版固定使用 AkShare 的东方财富相关接口：

- 行情数据：`stock_zh_a_spot_em()` 或 `stock_sh_a_spot_em()`。
- 历史日线：`stock_zh_a_hist()`。
- 资金流：`stock_individual_fund_flow()`。

如 AkShare 接口不可用，再切换到 Tushare Pro。切换数据源时必须在周报中记录数据源名称、接口名称和数据限制，避免不同平台的主力资金流算法混用。

参考文档：

- AkShare 股票数据文档：https://akshare.akfamily.xyz/data/stock/stock.html
- Tushare Pro 股票基础信息：https://tushare.pro/document/2?doc_id=25
- Tushare Pro A 股日线行情：https://tushare.pro/document/2?doc_id=27
- Tushare Pro 个股资金流向：https://tushare.pro/document/2?doc_id=170

## 10. 用户操作流程

### 10.1 初始化配置

用户检查或修改配置文件：

```text
config/stocks.yaml
```

配置内容包括：

- 股票代码。
- 公司简称。
- 交易所标识，如 `sh`。
- 是否启用跟踪。
- 行业备注。

### 10.2 采集每日数据

用户运行命令：

```bash
python scripts/fetch_daily.py
```

工具执行：

1. 读取股票池。
2. 调用 AkShare 行情接口。
3. 获取行情和资金流数据。
4. 清洗字段并统一字段名。
5. 保存或更新本地 CSV。

### 10.3 生成 Excel 周报

用户运行命令：

```bash
python scripts/generate_weekly_report.py --start 2026-06-08 --end 2026-06-12 --format xlsx
```

工具执行：

1. 读取指定日期范围内的历史数据。
2. 按股票分组。
3. 计算本周涨跌幅。
4. 汇总主力净流入。
5. 生成排名、异常说明和简要结论。
6. 输出 Excel 周报。

输出文件路径：

```text
reports/weekly_report_YYYY-MM-DD_YYYY-MM-DD.xlsx
```

## 12. Excel 周报格式要求

第一版明确以 Excel `.xlsx` 格式生成周报，不再以 Markdown 作为主输出格式。

### 12.1 文件命名

```text
weekly_report_2026-06-08_2026-06-12.xlsx
```

### 12.2 `周报摘要` 工作表

字段建议：

| 项目 | 内容示例 |
|---|---|
| 报告周期 | 2026-06-08 至 2026-06-12 |
| 跟踪股票数量 | 12 |
| 成功采集交易日数量 | 5 |
| 数据源 | AkShare / Eastmoney |
| 本周涨幅最高 | 中国出版，+3.20% |
| 本周跌幅最高 | 新华传媒，-2.10% |
| 主力净流入最高 | 凤凰传媒，1200 万元 |
| 主力净流出最高 | 城市传媒，-800 万元 |
| 结论摘要 | 本周出版传媒样本公司表现分化，资金流向集中于少数公司。 |
| 免责声明 | 本报告仅用于课程实践和数据观察，不构成投资建议。 |

### 12.3 `个股行情汇总` 工作表

字段建议：

| 字段 | 说明 |
|---|---|
| 股票代码 | 例如 601949 |
| 股票名称 | 中国出版 |
| 周初开盘价 | 日期范围内首个交易日开盘价 |
| 周末收盘价 | 日期范围内最后一个交易日收盘价 |
| 周最高价 | 本周最高价 |
| 周最低价 | 本周最低价 |
| 周涨跌幅 | `(周末收盘价 - 周初开盘价) / 周初开盘价` |
| 本周成交额合计 | 日期范围内成交额求和 |
| 数据完整性 | 完整 / 缺失 N 天 |

### 12.4 `资金流排名` 工作表

字段建议：

| 字段 | 说明 |
|---|---|
| 排名 | 按本周主力净流入合计降序排列 |
| 股票代码 | 股票代码 |
| 股票名称 | 公司简称 |
| 本周主力净流入合计 | 日期范围内主力净流入求和 |
| 本周主力净流入均值 | 日期范围内主力净流入平均值 |
| 净流入天数 | 主力净流入大于 0 的交易日数量 |
| 净流出天数 | 主力净流入小于 0 的交易日数量 |
| 连续状态 | 连续净流入 / 连续净流出 / 无明显连续 |

### 12.5 `每日明细` 工作表

该表保留每日清洗后的明细数据，字段与 `data/daily_stock_data.csv` 基本一致，便于用户核对。

### 12.6 `异常与缺失` 工作表

记录以下情况：

- 指定日期范围内某只股票缺少数据。
- 接口请求失败或返回空数据。
- 非交易日导致无数据。
- 停牌或异常波动。
- 主力资金流字段无法获取，只能使用替代口径。

### 12.7 `数据源说明` 工作表

必须写入：

- 数据源名称：AkShare。
- 实际接口名称：如 `stock_zh_a_spot_em`、`stock_zh_a_hist`、`stock_individual_fund_flow`。
- 数据获取时间。
- 主力资金流字段口径。
- 免费数据源可能存在延迟、缺失、字段变化或访问限制。
- 免责声明：本报告仅用于课程实践、数据整理和趋势观察，不构成任何投资建议。

### 12.8 Excel 样式要求

- 每个工作表首行加粗。
- 冻结首行。
- 启用自动筛选。
- 金额字段使用千分位格式。
- 涨跌幅字段使用百分比格式。
- 主力净流入为正数可用红色字体，为负数可用绿色字体。
- 自动调整列宽，避免内容显示不完整。

## 14. 模块设计

### 14.1 config_loader.py

职责：

- 读取股票池配置。
- 过滤 `enabled: true` 的股票。
- 校验股票代码、名称、市场标识字段是否存在。

### 14.2 data_fetcher.py

职责：

- 调用 AkShare 或 Tushare Pro 接口。
- 获取开盘价、收盘价、涨跌幅、主力净流入。
- 统一字段格式。
- 处理接口异常和缺失数据。

### 14.3 data_storage.py

职责：

- 保存每日数据到 CSV。
- 避免同一股票同一交易日重复写入。
- 读取指定日期范围内的历史数据。

### 14.4 analyzer.py

职责：

- 计算周涨跌幅。
- 汇总本周成交额。
- 汇总本周主力净流入。
- 生成主力净流入排名。
- 标记连续净流入或连续净流出。
- 生成异常与缺失数据清单。

### 14.5 excel_report_writer.py

职责：

- 将分析结果写入 Excel。
- 创建 `周报摘要`、`个股行情汇总`、`资金流排名`、`每日明细`、`异常与缺失`、`数据源说明` 工作表。
- 设置表头样式、冻结首行、自动筛选、列宽和数字格式。
- 写入数据源说明和投资风险提示。

## 15. 错误处理设计

| 场景 | 处理方式 |
|---|---|
| 配置文件不存在 | 输出明确错误，提示创建 `config/stocks.yaml` |
| 股票代码格式错误 | 跳过该股票并记录错误 |
| 数据源请求失败 | 重试一次，仍失败则记录失败原因 |
| 某只股票数据缺失 | 在 Excel 的 `异常与缺失` 工作表中记录 |
| 非交易日无数据 | 提示当前日期可能不是交易日，不写入空数据 |
| CSV 文件不存在 | 自动创建文件和表头 |
| 重复采集同一交易日数据 | 第一版建议跳过并提示 |
| 周报日期范围无数据 | 终止生成并提示用户检查日期 |
| Excel 文件正在被打开 | 提示关闭文件后重新生成 |

## 16. 验收标准

第一版完成标准：

1. 项目根目录存在完整 `design_updated.md`。
2. 可以通过配置文件维护出版传媒上市公司股票池。
3. 可以接通 AkShare API，并按股票代码获取特定股票行情信息。
4. 可以运行每日采集脚本，并生成或更新 `data/daily_stock_data.csv`。
5. CSV 中包含交易日期、股票代码、股票名称、开盘价、收盘价、涨跌幅、主力净流入等字段。
6. 可以指定日期范围生成 Excel 周报。

8. Excel 周报包含数据源说明和免责声明。
9. 数据缺失或接口失败时，程序不直接崩溃，而是输出明确提示并写入异常记录。
10. README 能说明安装依赖、配置股票池、采集数据和生成 Excel 周报的方法。

## 17. 周报分析内容

Excel 周报应至少包含以下分析：

1. 本周跟踪股票数量。
2. 本周成功获取数据的交易日数量。
3. 每只股票周初开盘价、周末收盘价和周涨跌幅。
4. 每只股票本周成交额合计。
5. 每只股票本周主力净流入合计。
6. 主力净流入排名前三。
7. 主力净流出排名前三。
8. 连续净流入或连续净流出的公司。
9. 数据缺失、接口失败或异常波动说明。
10. 数据源和字段口径说明。
11. 免责声明。

示例结论风格：

```text
本周出版传媒样本公司整体表现分化。按主力净流入统计，部分公司获得资金持续流入，但个别公司出现明显净流出。由于主力资金流指标来自第三方数据源，分析结果仅用于学习和数据观察，不构成投资建议。
```

## 20. 项目一句话总结

本项目是一个基于 Python 的上证 A 股出版传媒上市公司行情与主力资金流跟踪工具，用于通过 AkShare 等股票行情 API 定期采集开盘价、收盘价、涨跌幅和主力净流入数据，并按周生成结构化 Excel 周报，帮助用户观察出版传媒板块的价格变化和资金流向。
