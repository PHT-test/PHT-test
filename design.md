# 上证 A 股出版传媒上市公司行情与主力资金流周报分析工具设计文档

## 1. 项目背景

中国 A 股市场中存在多家以出版、发行、传媒、文化内容服务为主营业务的上市公司。这类公司分布在出版传媒行业，既具有文化产业属性，也受到资本市场行情、行业政策、资金流向和市场情绪影响。

手动查看每家公司的开盘价、收盘价、主力流入、主力流出等数据效率较低，也不利于长期观察行业板块变化。本项目计划开发一个轻量级数据分析工具，持续跟踪上证 A 股出版传媒类上市公司的关键行情和资金流数据，并每周自动生成一份结构化分析报告。

本项目只用于课程实践、数据整理和趋势观察，不构成任何投资建议。

## 2. 目标用户

- 关注出版传媒行业上市公司表现的研究者或学习者。
- 希望定期整理 A 股出版传媒类公司行情数据的个人用户。
- 需要快速生成周度市场观察材料的内容分析人员。

## 3. 核心场景

用户希望每周查看一次上证 A 股出版传媒类上市公司的行情变化和资金流向。工具根据预设股票池获取每日数据，保存历史记录，并在每周生成一份 Markdown 或 Excel 格式的周报。

典型使用流程：

1. 用户在配置文件中确认需要跟踪的股票代码和名称。
2. 用户运行数据采集命令，工具获取最新交易日数据。
3. 工具保存开盘价、收盘价、主力流入、主力流出等字段。
4. 每周末或指定时间，用户运行周报生成命令。
5. 工具读取本周历史数据，计算涨跌幅、主力净流入、排名等指标。
6. 工具输出结构化周报，供用户查看、提交或继续编辑。

## 4. 产品形态

第一版采用 Python 本地数据分析工具形态，不开发复杂 Web 系统。

选择原因：

- Python 适合行情数据获取、清洗、统计和报告生成。
- 本项目第一版重点是数据跟踪和周报输出，不需要复杂交互界面。
- 本地脚本便于课程作业展示和后续扩展。
- 相比 Web 工具，开发成本更低，数据处理流程更清晰。

后续如果需要展示效果，可以扩展为 Streamlit 或 Flask Web 页面。

## 7. 数据字段

每日采集字段：

| 字段名 | 含义 | 示例 |
|---|---|---|
| trade_date | 交易日期 | 2026-06-12 |
| stock_code | 股票代码 | 601949 |
| stock_name | 公司简称 | 中国出版 |
| open_price | 开盘价 | 8.50 |
| close_price | 收盘价 | 8.72 |
| main_inflow | 主力流入 | 10000000 |
| main_outflow | 主力流出 | 8000000 |
| main_net_inflow | 主力净流入 | 2000000 |
| change_pct | 当日涨跌幅 | 2.15% |
| data_source | 数据来源 | akshare/eastmoney |
| created_at | 数据保存时间 | 2026-06-12 16:30:00 |

说明：

- 主力净流入 = 主力流入 - 主力流出。
- 主力流入和主力流出不是交易所官方原始字段，通常来自第三方行情平台的资金流算法。
- 第一版必须固定一个数据源口径，避免混用不同平台导致指标不可比。

## 8. 功能清单

## 9. 用户操作流程

### 9.1 初始化配置

用户检查或修改配置文件：

```text
config/stocks.yaml
```

配置内容包括：

- 股票代码。
- 公司简称。
- 是否启用跟踪。
- 行业备注。

### 9.2 采集每日数据

用户运行命令：

```bash
python scripts/fetch_daily.py
```

工具执行：

1. 读取股票池。
2. 调用数据源接口。
3. 获取行情和资金流数据。
4. 清洗字段。
5. 计算主力净流入。
6. 追加保存到本地 CSV。

### 9.3 生成周报

用户运行命令：

```bash
python scripts/generate_weekly_report.py --start 2026-06-08 --end 2026-06-12
```

工具执行：

1. 读取指定日期范围内的历史数据。
2. 按股票分组。
3. 计算本周涨跌幅。
4. 汇总主力流入、主力流出、主力净流入。
5. 生成排名和简要分析文字。
6. 输出 Markdown 周报。

## 12. 模块设计

### 12.1 config_loader.py

职责：

- 读取股票池配置。
- 过滤 `enabled: true` 的股票。
- 校验股票代码、名称字段是否存在。

### 12.2 data_fetcher.py

职责：

- 调用 AkShare 或其他数据源接口。
- 获取开盘价、收盘价、主力流入、主力流出。
- 统一字段格式。
- 处理接口异常和缺失数据。

### 12.3 data_storage.py

职责：

- 保存每日数据到 CSV。
- 避免同一股票同一交易日重复写入。
- 读取指定日期范围内的历史数据。

### 12.4 analyzer.py

职责：

- 计算主力净流入。
- 计算周涨跌幅。
- 汇总本周资金流。
- 生成主力净流入排名。
- 标记连续净流入或连续净流出。

### 12.5 report_writer.py

职责：

- 将分析结果写成 Markdown。
- 生成表格和简要说明。
- 加入数据源说明和投资风险提示。

## 13. 错误处理设计

| 场景 | 处理方式 |
|---|---|
| 配置文件不存在 | 输出明确错误，提示创建 `config/stocks.yaml` |
| 股票代码格式错误 | 跳过该股票并记录错误 |
| 数据源请求失败 | 重试一次，仍失败则记录失败原因 |
| 某只股票数据缺失 | 在周报中列入数据缺失说明 |
| 非交易日无数据 | 提示当前日期可能不是交易日，不写入空数据 |
| CSV 文件不存在 | 自动创建文件和表头 |
| 重复采集同一交易日数据 | 覆盖或跳过，第一版建议跳过并提示 |
| 周报日期范围无数据 | 终止生成并提示用户检查日期 |

## 15. 数据源和口径说明

本项目需要使用第三方数据源获取行情和资金流数据。第一版优先考虑 AkShare，因为它适合 Python 数据分析项目，并提供较多 A 股行情数据接口。

需要注意：

- 开盘价、收盘价属于标准行情字段。
- 主力流入、主力流出通常来自第三方平台计算，不同平台口径可能不同。
- 本项目第一版必须固定一个数据源口径，不混合多个平台的主力资金流数据。
- 如果主力流入和主力流出无法直接获取，可以使用数据源提供的主力净流入字段作为替代，并在周报中说明限制。
- 免费数据源可能存在接口变化、访问限制、字段变化或数据延迟。

## 16. 周报分析内容

周报应至少包含以下分析：

1. 本周跟踪股票数量。
2. 本周成功获取数据的交易日数量。
3. 每只股票周初开盘价、周末收盘价和周涨跌幅。
4. 每只股票本周主力流入合计。
5. 每只股票本周主力流出合计。
6. 每只股票本周主力净流入合计。
7. 主力净流入排名前三。
8. 主力净流出排名前三。
9. 数据缺失、接口失败或异常波动说明。
10. 免责声明。

示例结论风格：

```text
本周出版传媒样本公司整体表现分化。按主力净流入统计，部分公司获得资金持续流入，但个别公司出现明显净流出。由于主力资金流指标来自第三方数据源，分析结果仅用于学习和数据观察，不构成投资建议。
```
